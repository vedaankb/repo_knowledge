# Knowledge Repository

This is a PurnaOS knowledge repository containing processed artifacts from your source code repository.

## Structure

```
.
├── manifest.json          # Root manifest tracking repo state
├── commits/               # Git commit metadata
│   └── {sha}.json
├── chunks/                # Code chunks with embeddings
│   └── {sha}/
│       └── {file_hash}.json
├── skills/                # PR-based feature skills
│   └── pr-{number}.json
└── deleted/               # Deleted files tracking
    └── {sha}.json
```

## Artifacts

### manifest.json
Root manifest containing:
- Source repository information
- Latest published commit SHA
- Total counts (chunks, commits, skills)
- Per-file index for incremental updates

### commits/{sha}.json
Git commit metadata for provenance:
- Commit message, author, timestamp
- Parent commits
- Changed files with stats
- Auto-generated summary

### chunks/{sha}/{file_hash}.json
Code chunks with embeddings:
- File path and symbol name
- Code content
- Tree-sitter metadata (kind, language)
- Line numbers
- 768-dimensional embedding vector
- Content hash for deduplication

### skills/pr-{number}.json
PR-based feature skills capturing team conventions:
- PR title, body, author
- Changed files
- Merge information
- Auto-generated skill summary with embedding

### deleted/{sha}.json
Tracks files deleted in each commit for cleanup

## Usage

This repository is automatically maintained by the `purna` CLI tool running in your source repository.

To set up PurnaOS in your source repository:

1. Install purna CLI
2. Run `purna init` in your source repo
3. Configure `.purnaOS/config.yaml` to point to this knowledge repo
4. Run `purna install` to set up git hooks
5. Commit and push - artifacts will be published automatically

## Privacy & Security

- This repository contains **code content** and **embeddings**
- Ensure proper access controls if your source code is sensitive
- Never commit secrets, API keys, or credentials
- Use GitHub's visibility settings to match your source repo's privacy level

## Maintenance

Artifacts are organized by commit SHA for efficient delta updates. Old artifacts are retained for history Q&A capabilities.

To clean up old artifacts:
```bash
purna cleanup --before-date 2026-01-01
```
